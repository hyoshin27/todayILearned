#자바와 JUnit을 활용한 실용주의 단위 테스트

>2-Right-BICEP 무엇을 테스트할 것인가?

Right : 결과가 올바른가?

B(boundary conditions): 경계 조건은 맞는가? 우리가 마주치는 수많은 결함은 모서리 사례(해피케이스 반대)이므로 이런부분을 처리해야한다.

- 모호하고 일관성 없는 입력값 예) 특수문자, 2월 29일, 00시 00분 등등

- 잘못된 양식의 데이터

- 수치적 오버플로를 일으키는 계산

- 비거나 빠진값

- 이성적인 기댓값을 훨씬 벗어나는 값

- 중복허용이 안될때 중복값이 있는 경우

- 정렬 알고리즘에 이미 정렬된 입력값을 넣는지 정렬 알고리즘에 역순 데이터를 넣는지

- 시간 순이 맞지 않는 경우

- conformance(준수), ordering(순서), range(범위), reference(참조), existence(존재), cardinality(기수), time(절대적 혹은 상대적 시간)

I(inverse relationship) : 역 관계를 검사할 수 있는가?

- 덧셈은 뺄셈으로 곱셈은 나눗셈, 교차검사에서는 보어

C(cross check) : 다른 수단을 활용하여 교차 검사를 할 수 있는가?

E(error conditions) : 오류조건을 강제로 일어나게 할 수 있는가? 좋은 단위 테스트는 로직전체에 대한 커버리지 달성만 하는 것이 아니다.

- 메모리가 가득찰 때

- 디스크 공간이 가득 찰 때

- 벽시계 시간에 관한 문제들(클라이언트간 시차)

- 네트워크 가용성 및 오류들

- 시스템 로드

- 제한된 색상 팔레트

- 매우 높거나 낮은 비디오 해상도

P(performance characteristics) ; 성능 조건은 기준에 부합하는가?

-성능이 핵심 고려 사항이라면 JMeter 같은 도구를 사용할수도 있고 단위 수준 성능 측정에 관심이 많다면 JUnitPerf 같은 서드파티 도구를 찾아도 된다.
